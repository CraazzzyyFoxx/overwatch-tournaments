Design system · v1.0

Editorial Tactical

A dark, broadcast-grade system for a competitive Overwatch tournament platform. Premium editorial layout — air, hairline rules, big mixed-case headlines — carrying a mission-control voice: faint coordinate grid, monospace labels, oversized grotesk numbers.

01 · Air over boxes

Group with whitespace and hairlines first; reach for a bordered card only for dense data.

02 · Numbers are telemetry

Stats in grotesk + mono, tabular, treated like a broadcast stat overlay.

03 · Meaning over decoration

Color and the role spectrum encode data — never ornament. One teal accent leads.

Divisions render as icons only — never a text label. The emblem encodes tier + sub-tier; the name lives in alt / title for screen readers and hover. Icons come from the workspace division grid (8 tiers × 5 = 40 divisions), so a community can swap the entire set.

Every per-stat KPI tile is a button opening the lobby leaderboard preselected on that stat: all players ranked, stat chips to switch, medals for top-3, your row highlighted with rank + top-% line. Inverted stats (Deaths) rank ascending and say lower is better. Bar = value vs leader.

Low-sample gate

ZaryaLOW SAMPLE6 games · Win% — · vs avg —

Percentiles and vs-avg comparisons are hidden under 10 games — show an em dash with a title explaining the rule, plus the LOW SAMPLE badge. "Top 2%" over 3 games is noise, not signal.

Dates

Relative and short: 2d ago, with the absolute date in title. Never a bare uppercase 2D — it collides with the Draw letter.

Deep links

Screen, active tab, list filters, search text and the selected event live in the URL (hash in the prototype, searchParams in Next). A profile state pasted into Discord must reopen exactly as sent.

Share card

The Player card renders to a 1200×630 OG-size PNG on canvas (grid ground, spectrum hairline, Onest numerals, form chips, profile URL) — copied to the clipboard with a download fallback.

Popover accessibility

Every hover popover (hero stats, MVP breakdown) also opens on focus and tap: triggers get tabindex="0" + aria-label, tap outside closes, scroll hides. Hover is an enhancement, not the only path.

Decisions log

The binding calls made while iterating the profile prototype (v46–v48) — each is a rule, not a suggestion.

Display face = Onest. Cyrillic-native geometric grotesk; identical texture on Latin, Cyrillic and digits. Space Grotesk rejected — no Cyrillic subset. Real build: next/font with subsets:["latin","cyrillic"].

Results ≠ roles. --win/--loss/--draw colour results (scores, W–D–L, form chips, pips); --good/--mid/--bad colour quality (winrate %, deltas, impact bars; thresholds ≥60 / 50–59 / <50). They share hues with support/live/warm today but may diverge without touching roles or statuses.

Role markers. Icon + label only in Role split. Tables and rows: icon-only with the name in title. Meta lines (header, event subtitle): text-only. Never both outside Role split.

Hero avatars. Stack 30px, compact detail rows 24px, table-inline 26px, standalone 32px. Rendered via HeroImage / AvatarStack, collapsing to +N.

Never colour-only. W/L map squares carry letters; trend dots ring the podium and hollow the bottom bucket; result chips always carry W/L/D letterforms.

Percentile bars. Horizontal under the value, fuller = better. The old vertical tick needed a caption to explain itself — an encoding that needs a caption is a failed encoding.

One name per metric. "Closeness" everywhere (was Proximity in one place), with a title glossary on first use. No SR/MMR anywhere — the platform has none.

Quiet chrome. Provider chips in the header are neutral (coloured glyph only); Share and Compare carry equal visual weight; one teal leads.

  --brand:#25d3bd; --warm:#f4b455;
